# Администрирование базы данных через приложение

## Обзор

В приложении реализована система разграничения доступа на основе ролей SQL Server. Пользователи входят в систему под своими учетными записями, созданными в SQL Server, и получают доступ к функциям в соответствии с назначенной ролью.

## Роли пользователей

| Роль | Описание | Кто получает |
|------|----------|--------------|
| **Pharmacy_Admin** | Полный доступ ко всем функциям системы | Директор, Администратор |
| **Pharmacy_Manager** | Доступ к складу, заказам, инвентаризации, управление сотрудниками | Заведующий складом, Менеджеры |
| **Pharmacy_Doctor** | Доступ к рецептам, технологиям приготовления, производству | Главный фармацевт, Провизор, Технолог |
| **Pharmacy_Staff** | Базовый доступ к продаже, движению товаров | Фармацевт, Кладовщик |
| **Pharmacy_Registrar** | Доступ к приемам заказов, пациентам | Регистратор |

## Вход в систему

1. Запустите клиентское приложение
2. Введите **логин** (фамилия на латинице, например: `ivanov`)
3. Введите **пароль** (по умолчанию: `12345678`)
4. Нажмите "Войти"

## Управление сотрудниками

### Добавление нового сотрудника

1. Войдите под учетной записью с ролью **Pharmacy_Admin** или **Pharmacy_Manager**
2. Перейдите на вкладку **"Сотрудники"**
3. Нажмите кнопку **"Добавить"**
4. Заполните все необходимые данные в открывшемся окне:
   - Фамилия, Имя, Отчество (обязательные поля)
   - Должность (обязательное поле)
   - Отдел (выберите из списка или введите новый)
   - Дата приема, телефон, email
   - Паспортные данные
   - Оклад
   - Права доступа (Менеджер, Может подписывать документы, Активен)
5. Нажмите **"Сохранить"**
6. Система автоматически:
   - Создаст запись сотрудника в базе данных
   - Создаст учетную запись SQL Server с логином из фамилии (транслитерация)
   - Назначит роль на основе должности
   - Установит пароль по умолчанию: `12345678`
7. После сохранения вы увидите сообщение с логином и паролем для входа

### Редактирование сотрудника

1. Выберите сотрудника в таблице
2. Нажмите кнопку **"Редактировать"**
3. Измените необходимые данные в окне редактирования
4. Нажмите **"Сохранить"**
5. Роль SQL Server будет автоматически обновлена при изменении должности

### Удаление сотрудника

1. Выберите сотрудника в таблице
2. Нажмите кнопку **"Удалить"**
3. Подтвердите удаление
4. Система автоматически:
   - Удалит учетную запись SQL Server
   - Удалит запись сотрудника из базы данных

**Примечание:** Все операции добавления, редактирования и удаления сотрудников теперь выполняются полностью через интерфейс приложения без необходимости ручного выполнения SQL-скриптов.

## Администрирование пользователей

Перейдите на вкладку **"Администрирование"** для управления учетными записями:

### Доступные функции:

- **Просмотр списка пользователей** — отображает все учетные записи SQL Server, их роли и привязку к сотрудникам
- **Создать пользователя** — показывает инструкцию по созданию нового пользователя через скрипт
- **Удалить пользователя** — показывает инструкцию по удалению выбранного пользователя
- **Сбросить пароль** — сбрасывает пароль выбранного пользователя на значение по умолчанию (12345678)
- **Обновить** — обновляет список пользователей

## Скрипты SQL

### create_employee_login.sql
Создает новую учетную запись для сотрудника:
- Создает LOGIN на уровне сервера
- Создает USER в базе данных PharmacyDB
- Назначает роль
- Устанавливает пароль по умолчанию

### delete_employee_login.sql
Удаляет учетную запись сотрудника:
- Удаляет из всех ролей
- Удаляет USER из базы данных
- Удаляет LOGIN с уровня сервера

## Безопасность

- Пароли хранятся в SQL Server с использованием встроенных механизмов безопасности
- Каждая роль имеет строго определенные права доступа к таблицам и операциям
- Приложения подключается к БД под учетными данными вошедшего пользователя
- Все операции выполняются от имени конкретного пользователя с аудитом через SQL Server

## Примечания

1. **EmployeeID** — технический идентификатор, не используется для входа
2. **Логин** — формируется из фамилии на латинице (ivаnov, petrova и т.д.)
3. **Пароль по умолчанию** — `12345678` (рекомендуется менять после первого входа)
4. Для добавления новых фамилий в систему маппинга обновите словарь в `LoginViewModel.cs`
